# Claude Prompt：多平台分发

## 使用场景
当你完成了一篇内容，想把它改写/适配成多个平台的版本（公众号、小红书、X、抖音等）。

## Prompt 模板

```
@用Claude和Obsidian做内容生产工厂.md

帮我把这篇内容改写成多平台版本。

【原文来源】
- 文件路径：{{原文文件路径}}
- 或粘贴原文：
"""
[粘贴原文内容]
"""

【需要适配的平台】（可多选）
- [ ] 公众号长文（1500-3000字，故事性，深度）
- [ ] 小红书图文（800-1200字，分点清晰，干货密度高，CTA强）
- [ ] X/Twitter 推文串（8-15条，每条独立成推，递进结构）
- [ ] 抖音/视频口播脚本（1-3分钟，口语化，节奏感，视觉提示）
- [ ] 知乎回答（深度分析，引用数据，结构清晰）
- [ ] B站视频脚本（开场钩子+知识点+互动）

【改写要求】
- 语气风格：{{保持原文/调整}}
- 核心观点：{{保持不变/适当调整}}
- 需要埋的钩子：{{如有}}
- 其他特殊要求：

请执行以下操作：

1. 分析原文
   - 核心观点：这篇文章到底在讲什么
   - 目标受众：写给谁看的
   - 情绪价值：带来了什么感受
   - 记忆点：最可能记住的一句话

2. 为每个选中的平台生成适配版本
   - 公众号版：完整深度版，保留案例和细节
   - 小红书版：分页结构，每页一个知识点+金句+emoji
   - X版：串推结构，用 --- 分隔每条，控制长度
   - 视频版：分镜脚本格式，标注画面+台词+时长

3. 保存到对应工厂文件夹
   - 公众号版 → `05-图文工厂/公众号文章/`
   - 小红书版 → `05-图文工厂/X推文线程/`（或单独建小红书文件夹）
   - 视频脚本 → `04-视频工厂/对应平台/`
   - X版 → `05-图文工厂/X推文线程/`

4. 建立回链
   - 每个平台版本都链接回原文
   - 原文更新状态为「已多平台分发」
   - 在关联笔记中添加所有版本链接

完成后告诉我：
- 生成了几个平台的版本
- 各版本保存路径
- 各版本的核心差异（一句话概括）
- 建议的发布顺序
```

副标题：3步上手方法论｜告别焦虑

AI只是工具，决策权在你

**金句**：
"AI不是你的对手，是你的队友"

**行动**：
放下焦虑，开始尝试

## 第2页：第一步 - 学会提示词
**标题**：好的提示词=清晰的指令

**核心观点**：
不要只给主题，要给框架
告诉他角色、目标、约束、格式

**示例**：
❌ "写一篇关于AI的文章"
✅ "你是资深科技作者，请用3个案例说明AI如何提升写作效率，800字，口语化"

**金句**：
"提示词的质量，决定输出的质量"

---

## 第3页：第二步 - 结构化输出
**标题**：让AI按你的格式来

**核心观点**：
给AI一个模板，他会填得更好
大纲、段落、要点，都可以结构化

**示例模板**：
```
主题：___
核心观点：___
3个论据：
1. ___
2. ___
3. ___
结论：___
```

**金句**：
"结构是骨架，内容是血肉"

**标题**：第一稿只是开始

**迭代指令示例**：
- "这部分太长了，精简到100字"
- "加入一个具体案例"
- "语气更口语化一些"
- "换个角度，从反面论证"

**金句**：
"好文章是改出来的，好提示是试出来的"

**总结**：
AI写作3步法：
1️⃣ 学会提示词

AI写作不是魔法，是方法论。

分享一下我总结的「AI写作3步法」，让你从焦虑到掌控。

---

1/10

很多人觉得AI写作就是输入一个主题，等它出稿。

结果往往是：
- 内容空洞
- 语气生硬
- 完全不像人写的

问题在哪？

---

2/10

问题在于：你把AI当成了作者，但其实它只是个 amplifier（放大器）。

你输入模糊的想法 → 它输出模糊的废话
你输入清晰的指令 → 它输出清晰的内容

所以第一步是：学会写提示词（prompt）。

---

3/10

好的提示词 = 清晰的指令。

不要只说"写一篇关于AI的文章"。

试试这个结构：
- 角色：你是一位资深科技作者
- 目标：用3个案例说明AI如何提升写作效率
- 约束：800字，口语化，避免专业术语

---

4/10

第二步：结构化输出。

给AI一个模板，它会填得更好。

比如你可以说：
"请按这个结构写：
1. 痛点引入（1段）
2. 核心观点（1句）
3. 3个论据（每段1个案例）
4. 行动建议（2-3点）"

---

5/10

结构化有两个好处：

1. 你掌控节奏，AI填充内容
2. 后续修改更容易（"第3部分太长，精简到100字"）

记住：结构是骨架，内容是血肉。
没有好的骨架，血肉再丰满也立不住。

---

6/10

第三步：迭代优化。

很多人期待一次出稿，结果往往失望。

现实是：第一稿只是开始。

好的AI写作流程 = 生成 → 反馈 → 优化 → 再反馈 → 再优化

---

7/10

迭代时可以用这些指令：

- "这部分太长了，精简到100字"
- "加入一个具体案例，最好是名企的"
- "语气更口语化，像和朋友聊天"
- "换个角度，从反面论证这个观点"
- "加一个小标题，让结构更清晰"

---

8/10

AI写作的本质，不是让AI代替你思考，而是：

把你模糊的创意 → 变成清晰的文字
把你零散的想法 → 变成有结构的内容
把你第一稿的60分 → 迭代到80分

你依然是作者，AI是 amplifier。

---

9/10

总结一下「AI写作3步法」：

1️⃣ 学会提示词 → 给AI清晰的指令
2️⃣ 结构化输出 → 用模板掌控节奏
3️⃣ 迭代优化 → 从60分改到80分

掌握这三步，AI写作就从焦虑变成掌控。

---

10/10

最后一个小建议：

不要试图让AI一次性写出完美的稿子。

先让它写个60分的框架，
然后你在这个框架上修改、添加你的洞察、调整语气。

这样最高效，也最能体现你的风格。
