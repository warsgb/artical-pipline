# 内容工厂工作流规则

## 一、文件流转路径

```
灵感 → 选题 → 大纲 → 初稿 → 终稿 → 发布 → 归档 → 复盘
  ↓       ↓       ↓        ↓        ↓        ↓        ↓
01-灵感  02-选题 03-大纲  03-初稿  03-终稿  05/04  06-归档
```

## 二、必须字段（YAML Frontmatter）

所有文档必须有以下字段：

```yaml
---
title: "标题"
stage: "阶段名"
status: "状态"
create_date: "YYYY-MM-DD"
update_date: "YYYY-MM-DD"  # 最后更新时间
---
```

### stage 可选值
- `灵感` - 在灵感库
- `选题` - 在选题池
- `大纲` - 大纲阶段
- `初稿` - 正在写
- `终稿` - 已完稿
- `已发布` - 已发布
- `已归档` - 在归档区

### status 可选值
- `待处理` / `待写` / `待完善`
- `进行中` / `写作中`
- `已通过` / `已完成`
- `已转化`（灵感专用）
- `终稿待发布`（终稿专用）
- `已发布`
- `已归档`

### 可选字段
- `advance_note` - 推进备注，记录状态变更��因
- `source_topic` - 来源选题文件（大纲/初稿使用）
- `source_outline` - 来源大纲文件（初稿使用）
- `archive_date` - 归档时间
- `publish_date` - 发布时间
- `publish_platform` - 发布平台

## 三、回链规则

每篇文档必须在文末添加「关联笔记」：

```markdown
## 关联笔记
- 灵感来源：[[01-灵感与素材库/具体文件]]
- 选题来源：[[02-选题池/具体文件]]
- 规则参考：[[90-系统配置/规则/]]
```

最少3条回链。

## 四、命名规范

### 文件命名
- 日期优先：`YYYY-MM-DD-标题关键词.md`
- 例如：`2026-03-07-AI写作方法论.md`

### 标签使用
- 大分类用文件夹
- 细分类用标签 `#标签名`
- 状态用标签 `#待写` `#进行中` `#已发布`

## 五、状态流转矩阵

| 当前阶段 | 当前status | 触发动作 | 更新后stage | 更新后status | 是否创建下游 |
|---------|-----------|---------|------------|------------|------------|
| 灵感 | 任意 | 推进 | 灵感 | 已转化 | ❌ |
| 选题 | 待写 | 推进 | 选题 | 进行中 | ❌ |
| 选题 | 进行中 | 提交修改 | 选题 | 待完善 | ❌ |
| 选题 | 待完善 | 通过 | 选题 | 已通过 | ❌ |
| 选题 | 已通过 | 继续 | 大纲 | 待完善 | ✅ 创建大纲 |
| 大纲 | 待完善 | 完成 | 大纲 | 已通过 | ❌ |
| 大纲 | 已通过 | 继续 | 初稿 | 写作中 | ✅ 创建初稿 |
| 初稿 | 写作中 | 提交修改 | 初稿 | 待完善 | ❌ |
| 初稿 | 待完善 | 通过 | 初稿 | 已通过 | ❌ |
| 初稿 | 已通过 | 继续 | 终稿 | 终稿待发布 | ✅ 创建终稿 |
| 终稿 | 终稿待发布 | 发布 | 终稿 | 已发布 | ❌ |
| 终稿 | 已发布 | 归档 | 已归档 | 已归档 | ✅ 触发归档流程 |

## 六、流转触发语（对Claude说）

| 场景 | 触发语 | Claude动作 |
|------|--------|------------|
| 存灵感 | 「把这条存进内容工厂」 | 创建灵感卡+生成3选题 |
| 选选题 | 「从待写选题里挑N个」 | 读取选题池+推荐+理由 |
| 写大纲 | 「给这个选题写大纲」 | 生成大纲+存到03-内容工厂 |
| 生成内容 | 「按大纲生成XXX稿」 | 生成文章+存到对应工厂 |
| 多平台 | 「改成X推文/小红书」 | 改写+存到05-图文工厂 |
| 发布归档 | 「标记已发布」 | 移动到06-归档+创建复盘卡 |

### 新增自动化命令

| 命令 | 含义 | 示例 |
|------|------|------|
| `/推进` | 推进到下一阶段 | `/推进` |
| `/通过` | 审核通过 | `/通过` |
| `/退回` | 退回修改 | `/退回` |
| `/发布` | 发布内容 | `/发布 公众号` |
| `/归档` | 归档内容链 | `/归档` |
| `/状态` | 查看状态 | `/状态` |

### 触发语示例

**推进内容：**
- "推进"
- "推进到下一步"
- "继续"
- "开始写这个选题"
- "/推进"

**归档内容：**
- "归档这个内容"
- "内容已完成，归档"
- "/归档"

**查看状态：**
- "查看状态"
- "这个内容进行到哪了"
- "进度如何"
- "/状态"

## 七、日常检查清单

每天开始工作前：

- [ ] 检查 `02-选题池/待写选题库` 有没有待处理的
- [ ] 检查 `03-内容工厂/` 有没有进行中的草稿
- [ ] 看一下 `01-灵感与素材库` 有没有新存的灵感需要处理

每周结束时：

- [ ] 把已完成的移动到 `06-已发布归档`
- [ ] 完成复盘卡片
- [ ] 整理本周的选题池，删除过期的
