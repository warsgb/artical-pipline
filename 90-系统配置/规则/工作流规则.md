# 内容工厂工作流规则

## 一、文件流转路径

```
灵感 → 选题 → 大纲 → 初稿 → 终稿 → 发布 → 归档 → 复盘
  ↓       ↓       ↓        ↓        ↓        ↓        ↓
01-灵感  02-选题 03-大纲  03-初稿  03-终稿  05/04  06-归档
```

## 二、必须字段（YAML Frontmatter）

所有文档必须有以下字段：

```yaml
---
title: "标题"
stage: "阶段名"
status: "状态"
create_date: "YYYY-MM-DD"
---
```

### stage 可选值
- `灵感` - 在灵感库
- `选题` - 在选题池
- `大纲` - 大纲阶段
- `初稿` - 正在写
- `终稿` - 已完稿
- `已发布` - 已发布
- `已归档` - 在归档区

### status 可选值
- `待处理`
- `进行中`
- `待完善`
- `已完成`
- `已发布`
- `已归档`

## 三、回链规则

每篇文档必须在文末添加「关联笔记」：

```markdown
## 关联笔记
- 灵感来源：[[01-灵感与素材库/具体文件]]
- 选题来源：[[02-选题池/具体文件]]
- 规则参考：[[90-系统配置/规则/]]
```

最少3条回链。

## 四、命名规范

### 文件命名
- 日期优先：`YYYY-MM-DD-标题关键词.md`
- 例如：`2024-03-07-AI写作方法论.md`

### 标签使用
- 大分类用文件夹
- 细分类用标签 `#标签名`
- 状态用标签 `#待写` `#进行中` `#已发布`

## 五、流转触发语（对Claude说）

| 场景 | 触发语 | Claude动作 |
|------|--------|------------|
| 存灵感 | 「把这条存进内容工厂」 | 创建灵感卡+生成3选题 |
| 选选题 | 「从待写选题里挑N个」 | 读取选题池+推荐+理由 |
| 写大纲 | 「给这个选题写大纲」 | 生成大纲+存到03-内容工厂 |
| 生成内容 | 「按大纲生成XXX稿」 | 生成文章+存到对应工厂 |
| 多平台 | 「改成X推文/小红书」 | 改写+存到05-图文工厂 |
| 发布归档 | 「标记已发布」 | 移动到06-归档+创建复盘卡 |

---

## 六、日常检查清单

每天开始工作前：

- [ ] 检查 `02-选题池/待写选题库` 有没有待处理的
- [ ] 检查 `03-内容工厂/` 有没有进行中的草稿
- [ ] 看一下 `01-灵感与素材库` 有没有新存的灵感需要处理

每周结束时：

- [ ] 把已完成的移动到 `06-已发布归档`
- [ ] 完成复盘卡片
- [ ] 整理本周的选题池，删除过期的
