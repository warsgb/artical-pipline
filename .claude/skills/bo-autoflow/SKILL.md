# 自动流水线 Skill

这是内容生产工厂的自动化编排技能：分析用户输入，自动串联 bo-1 到 bo-7 的技能执行，实现一键生成完整内容并发布归档。

## ⚠️ 重要：文件路径规范（使用本技能前必读）

### 正确的目录结构

所有内容工厂的文件**必须**遵循以下标准路径结构：

```
D:/obs/social/
├── 00-网文收藏/                    # 原始文章备份
├── 01-灵感与素材库/1-日常灵感剪报/  # 灵感卡片
├── 02-选题池/待写选题库/          # 选题文件
├── 03-内容工厂/                    # 生产中的内容 ✅ 正确
│   ├── 1-大纲对焦区/              # 大纲文件
│   ├── 2-初稿打磨区/              # 初稿文件
│   └── 3-终稿发布区/              # 终稿文件
├── 04-视频工厂/                    # 视频脚本
├── 05-图文工厂/                    # 分发版本
└── 06-已发布归档/                 # 已发布内容
```

### ❌ 绝对禁止的错误路径

**错误示例**：`内容工厂/03-内容工厂/1-大纲对焦区/`

**正确路径**：`03-内容工厂/1-大纲对焦区/`

**问题说明**：路径中出现了重复的"内容工厂"目录名（`内容工厂/` + `03-内容工厂/`）。**正确的路径应该以 `03-` 开头，而不是 `内容工厂/`。**

### 路径验证检查清单

在写入任何文件前，必须确认：

- [ ] 路径以 `03-内容工厂/` 开头，而不是 `内容工厂/03-内容工厂/`
- [ ] 路径中没有重复的目录名
- [ ] 使用正斜杠 `/` 而不是反斜杠 `\`
- [ ] 文件名格式：`YYYY-MM-DD-标题-阶段.md`

### 文件命名规范

1. **日期格式**：`YYYY-MM-DD-`
2. **文件标题**：使用短横线连接，如 `OpenClaw创始人教你如何管理AI Agent`
3. **阶段标识**：
   - 大纲文件：`-大纲`
   - 初稿文件：`-初稿`
   - 终稿文件：`-终稿`
4. **完整示例**：`2026-03-10-OpenClaw创始人教你如何管理AI Agent-终稿.md`

## 两种执行模式

### 模式1：交互模式（默认）
- 每个阶段需要用户确认
- 用户可以调整选项
- 适合需要精细控制的场景

### 模式2：全自动模式
- 用户给一个URL或话题，自动跑完整个流程
- 自动选择最佳选题
- 自动选择目标平台
- **中间无需任何用户干预**
- 适合快速生成内容的场景

## 触发条件

当用户出现以下任一情况时，触发此skill：

**交互模式触发：**
- 明确说"自动写一篇"、"帮我写完整的"
- 说"从灵感到终稿"
- 提供素材内容后说"开始写"

**全自动模式触发：**
- 说"全自动"、"全自动跑完"、"不需要问我"
- 说"一键生成"、"直接生成"
- 提供URL后说"直接写"、"帮我写完"
- 输入格式：`/自动 https://xxx` 或 `自动写一篇 https://xxx`

### 1. 全自动模式详解

当用户触发全自动模式时，整个流程无需用户干预：

**触发关键词：**
- "全自动"、"全自动跑完"、"不需要问我"
- "一键生成"、"直接生成"
- "直接写"、"帮我写完"
- `/自动 https://xxx`（命令格式）

**全自动执行流程：**

1. **步骤1：抓取URL内容**
   - 自动使用 chrome-dev MCP 抓取网页
   - 提取标题、正文、核心观点

2. **步骤2：自动创建灵感卡**
   - 提取2-4个核心亮点
   - 自动生成标题

3. **步骤3：自动选择最佳选题**
   - 分析3个选题的质量
   - **自动选择最合适的选题**（标准见下文）
   - 无需用户确认

4. **步骤4：自动生成大纲**
   - 基于选题自动生成大纲
   - 无需用户确认

5. **步骤5：自动生成初稿**
   - 基于大纲自动填充内容
   - 无需用户确认

6. **步骤6：自动生成终稿**
   - 润色、优化、精简
   - 无需用户确认

7. **步骤7：自动生成分发版本（可选）**
   - 根据内容类型自动选择1-2个平台
   - 自动生成小红书版本（如内容适合）
   - bo-5-distribute 会自动调用 bo-autoimages 生成配图

8. **步骤8：自动发布到平台（可选）**
   - 根据分发版本自动调用 bo-6-publish
   - 公众号版本 → bo-6-publish 会调用 bo-x-pubwechat 发布到草稿箱
   - 小红书版本 → 提示手动发布（待自动化）
   - 无需用户确认

9. **步骤9：自动归档（可选）**
   - 内容发布完成后，自动调用 bo-7-archive
   - 创建归档索引和复盘卡片
   - 无需用户确认

**自动选择最佳选题的算法：**

```
选题评分标准：
1. 相关度（与原文主题的契合度）：40%
2. 受众广度（目标读者群体大小）：30%
3. 话题性（是否有传播潜力）：20%
4. 可操作性（是否能写出深度）：10%

选择得分最高的选题
```

**自动选择目标平台的算法：**

```
平台选择标准：
- 如果内容是"干货/教程类" → 小红书
- 如果内容是"热点/观点类" → 公众号 + 微博
- 如果内容是"故事/案例类" → 公众号 + 知乎
- 如果内容是"短快/趣味类" → 抖音 + 小红书
- 默认：生成小红书版本
```

### 2. 智能路径判断

### 1. 智能路径判断

根据用户输入的内容和需求，自动判断应该执行哪些阶段：

| 用户输入类型 | 建议路径 |
|-------------|---------|
| 只有话题/想法 | 1→2→3→4→5→6→7（灵感→选题→大纲→初稿→终稿→分发→发布→归档） |
| 有素材内容 | 1→2→3→4→5→6→7（灵感入库→选题→大纲→初稿→终稿→分发→发布→归档） |
| 有URL/文章链接 | 1→2→3→4→5→6→7（抓取→选题→大纲→初稿→终稿→分发→发布→归档） |
| 只要选题 | 1→2（灵感→选题） |
| 只要初稿 | 2→3（选题→大纲→初稿） |
| 只要终稿 | 3→4（大纲→初稿→终稿） |

### 2. 交互式确认

自动判断后，向用户确认执行路径：

```
📋 自动分析结果：

检测到您提供了：{{输入类型}}
建议执行路径：{{路径}}

请确认：
1. 执行完整流程（灵感→选题→大纲→初稿→终稿→分发→发布→归档）
2. 执行到分发（灵感→选题→大纲→初稿→终稿→分发）
3. 执行到终稿（灵感→选题→大纲→初稿→终稿）
4. 只执行到选题
5. 只执行到初稿
6. 自定义路径
```

### 3. 自动执行

用户确认后，自动串联执行相应技能：
- 每个阶段完成后，自动进入下一阶段
- 传递上下文和数据
- 汇总最终结果

## 工作流程

### 步骤1：分析用户输入

**检查输入类型：**

1. **URL输入**（如 `https://...`）
   - 自动触发 bo-1-inspiration 的 URL 抓取功能

2. **素材内容**（如文本、截图描述、金句等）
   - 自动触发 bo-1-inspiration

3. **明确指定**（如"写一篇关于xxx的文章"）
   - 直接从 bo-2 开始

### 步骤2：判断执行路径

根据分析结果，给出路径建议：

```
📋 路径分析

输入类型：{{URL/素材/话题}}
建议路径：{{1→2→3→4 / 1→2 / 2→3 / 3→4}}
```

### 步骤3：模式判断与执行

**判断执行模式：**

```
检测关键词：
- 如果包含"全自动"、"不需要问我"、"一键生成"、"直接" → 全自动模式
- 否则 → 交互模式
```

**交互模式：**
- 询问用户确认执行路径
- 用户选择后执行

**全自动模式：**
- 直接开始执行，无需任何确认
- 自动选择最佳选项

### 步骤4：执行技能链

**执行过程中：**

1. **阶段1：灵感入库（bo-1）**
   - 如果输入是URL：抓取内容
   - 创建灵感卡
   - 生成3个选题

2. **阶段2：选题转大纲（bo-2）**
   - 用户选择1个选题
   - 创建大纲

3. **阶段3：大纲转初稿（bo-3）**
   - 基于大纲生成初稿

4. **阶段4：初稿转终稿（bo-4）**
   - 润色优化生成终稿

5. **阶段5：多平台分发（bo-5）**
   - 可选：根据用户需求生成分发版本

### 步骤5：汇总报告

完成后，生成汇总报告：

```
✅ 自动流水线执行完成！

📊 执行摘要：
- 执行路径：{{1→2→3→4}}
- 总耗时：约{{X}}分钟
- 生成内容：
  - 选题：{{X}}个
  - 大纲：{{X}}个
  - 初稿：{{X}}篇
  - 终稿：{{X}}篇
  - 分发版本：{{X}}个

📁 文件位置：
- 灵感卡：{{路径}}
- 选题：{{路径}}
- 大纲：{{路径}}
- 初稿：{{路径}}
- 终稿：{{路径}}

📌 下一步：
- 查看终稿内容
- 使用 /分发 创建更多平台版本
- 使用 /归档 归档内容链
```

## 智能判断规则

### 输入类型识别

| 特征 | 类型判断 |
|------|---------|
| 包含 `http://` 或 `https://` | URL |
| 包含 ">、"或引用格式 | 素材内容 |
| 包含"关于"、"关于xxx的文章" | 话题 |
| 包含文件路径 | 续写/改写 |

### 路径推荐规则

```
IF 输入是URL:
    推荐 完整流程 (1→2→3→4)
ELIF 输入是长文本:
    推荐 完整流程 (1→2→3→4)
ELIF 输入是短话题:
    推荐 到初稿 (2→3→4)
ELIF 用户说"只要选题":
    推荐 到选题 (1→2)
ELIF 用户说"帮我写":
    推荐 完整流程 (1→2→3→4)
```

### 路径验证方法

所有技能脚本在写入文件前，必须调用路径验证工具：

```javascript
// 路径验证函数
function validatePath(inputPath) {
  // 检测常见问题：路径中包含重复的目录名
  if (inputPath.includes('内容工厂/') && inputPath.includes('03-内容工厂')) {
    // 修复重复路径
    return inputPath.replace(/内容工厂\//, '');
  }
  return inputPath;
}
```

## 注意事项

### 交互模式
1. **用户确认**：自动判断后必须用户确认执行路径
2. **灵活选择**：用户可以跳过不需要的阶段
3. **上下文传递**：每个阶段的输出自动作为下一阶段的输入

### 全自动模式
1. **无需确认**：检测到全自动关键词后，直接开始执行
2. **自动选择**：选题、平台等自动选择最佳选项
3. **快速执行**：跳过所有交互环节，节省时间

### 通用
4. **错误处理**：某个阶段失败时，询问用户是继续还是回退
5. **可中断**：用户可以说"暂停"、"停止"中断执行
