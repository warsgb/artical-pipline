# 多平台分发 Skill

这是内容生产工厂的第5阶段：将终稿内容改编为适合不同平台的版本，实现一内容多平台分发。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"/分发"、"多平台分发"、"改编为平台名"
- 说"创建小红书版本"、"发布到知乎"
- 说"改成推文"、"做成短视频脚本"
- 对一个终稿文件说"/分发 平台1 平台2"

**前置条件：**
- 应该有一个已发布或待发布的终稿内容
- 用户指定目标平台（如果没有指定，将根据内容类型自动判断）

## 支持的平台

### 文字平台
1. **公众号** - 深度长文，2000-3000字
2. **知乎** - 问答形式，1500-2500字
3. **小红书** - 图文笔记，800-1000字
4. **微博** - 短内容，300-500字
5. **即刻** - 中短内容，500-800字
6. **Medium** - 英文深度，2000-3000字

### 视频平台
7. **抖音** - 短视频脚本，60-90秒
8. **B站** - 中视频脚本，5-10分钟
9. **视频号** - 短视频脚本，60-90秒

### 其他平台
10. **播客** - 音频脚本，10-20分钟
11. **Newsletter** - 邮件内容，1500字左右

## 工作流程

### 步骤0：判断目标平台（重要！）

**如果用户没有明确指定平台**，根据内容类型自动判断：

| 内容类型 | 自动选择平台 | 说明 |
|---------|------------|------|
| 干货/教程类 | 小红书 + 公众号 | 实用性强，适合多平台 |
| 热点/观点类 | 公众号 + 小红书 | 公众号深度 + 小红书传播 |
| 故事/案例类 | 公众号 + 知乎 | 公众号传播 + 知乎沉淀 |
| 短快/趣味类 | 小红书 + 抖音 | 视觉化传播 |
| 对比分析类 | 小红书 + 知乎 | 表格对比适合小红书 + 知乎深度 |

**判断逻辑：**
1. 读取终稿的 `content_type` 字段
2. 根据上表自动选择目标平台
3. 默认至少创建 **小红书 + 公众号** 两个版本（除非用户明确指定不想要）

### 步骤1：读取源内容

**操作：**
1. 读取用户指定的终稿文件
2. 提取内容的核心要素：
   - 核心观点
   - 关键案例
   - 金句
   - 数据支撑
   - 行动建议

### 步骤2：分析目标平台特点

**分析维度：**
1. **内容形式：** 长文/短文/图文/视频/音频
2. **字数限制：** 目标字数范围
3. **用户特点：** 年龄、兴趣、阅读习惯
4. **算法偏好：** 什么样的内容更容易被推荐
5. **格式要求：** 标题、封面、标签、排版

### 步骤3：创建平台适配版本

**操作：** 根据目标平台特点，将源内容改编为适合该平台的版本

**文件位置：**
- 文字/图文平台：`05-图文工厂/{{平台名}}/YYYY-MM-DD-{{title}}-{{平台名}}.md`
- 视频平台：`04-视频工厂/{{平台名}}/YYYY-MM-DD-{{title}}-{{平台名}}.md`

**使用模板：** 根据平台选择合适的模板

**填充字段：**
```yaml
---
title: "{{适配后的标题}}"
stage: 已发布
content_type: "{{内容类型}}"
platform: "{{平台名称}}"
status: 待发布/已发布
source_content: "[[原终稿文件路径]]"
create_date: "{{DATE:YYYY-MM-DD}}"
update_date: "{{DATE:YYYY-MM-DD}}"
word_count: "预计{{字数}}字"
tags: ["{{标签1}}", "{{标签2}}", "{{标签3}}"]
advance_note: "从「{{原标题}}」改编而来"
---
```

### 步骤3.5：自动生成配图并插入到文章（关键步骤）

> **重要提示：** 对于小红书和公众号版本，配图生成是**必需步骤**，不是可选步骤。必须等待 bo-autoimages 完全执行完成后，才能结束整个分发流程。

> **触发条件：** 仅小红书和公众号版本自动生成配图，其他平台（知乎、微博、抖音等）不自动生成

**操作：** 在创建平台适配版本后，**调用 bo-autoimages 生成配图，等待图片生成完成后，将图片引用插入到 markdown 文件中**

**关键原则：**
1. **同步等待**：bo-5-distribute 必须等待 bo-autoimages 完全执行完成，获取到图片路径列表后才能继续
2. **阻塞执行**：图片生成未完成前，不能结束 bo-5-distribute 流程
3. **完整性检查**：确保所有图片都已成功生成，才能认为分发流程完成

**执行流程：**

```
1. 判断平台是否需要自动生成配图
   - 小红书：✅ 自动生成（3:4比例）
   - 公众号：✅ 自动生成（4:3比例）
   - 其他平台：⏭️ 跳过，继续步骤4

2. 调用 bo-autoimages 生成配图（阻塞等待）
   /bo-autoimages --file "刚创建的适配版本文件路径"

   ⚠️ 重要：必须等待 bo-autoimages 完全执行完成，返回结果后才能继续

   bo-autoimages 会自动：
   - 分析内容类型和关键点
   - 判断合适的图片数量（3-6张）
   - 选择正确的比例（小红书3:4，公众号4:3）
   - 输出到 05-图文工厂/{平台名}/img-{标题}/
   - **返回图片路径列表**（JSON格式）

3. 接收 bo-autoimages 的返回值（同步获取）
   从返回值中提取图片路径列表：
   {
     "success": true,
     "output_dir": "05-图文工厂/小红书/img-{标题}/",
     "images": [
       {"path": "img-{标题}/p1-cover.png", "alt_text": "封面图"},
       {"path": "img-{标题}/p2-xxx.png", "alt_text": "xxx"},
       ...
     ]
   }

   ⚠️ 如果没有获取到返回值或返回失败，必须重试或报错，不能继续

4. 将图片引用插入到 markdown 文件中（核心步骤）
   使用 Edit 工具，在合适的位置插入图片引用：

   封面图：插入到标题之后
   ![封面图](img-{标题}/p1-cover.png)

   内容图：根据内容结构插入到对应段落
   ![图片说明](img-{标题}/p2-xxx.png)

   结尾图：插入到CTA之前
   ![行动建议](img-{标题}/pN-action.png)

5. 更新文件的 frontmatter
   在 frontmatter 中添加 images 字段记录图片路径：
   ---
   images:
     - img-{标题}/p1-cover.png
     - img-{标题}/p2-xxx.png
     - ...
   ---

6. 验证图片生成和插入（最终检查）
   ✅ 检查图片文件是否真实存在于输出目录
   ✅ 检查图片引用路径是否正确
   ✅ 检查图片是否能在 Obsidian 中正常显示
   ✅ 确认所有必需图片都已生成

   只有通过以上所有检查，才能认为配图生成步骤完成
```

**错误处理：**
- 如果 bo-autoimages 调用失败或超时，必须重试（最多3次）
- 如果重试后仍失败，向用户报告错误，询问是否继续（不配图）或中止流程
- 绝不能忽略配图生成失败的情况，继续结束流程

**关键要点：**
1. **bo-autoimages 只负责生成图片**，返回图片路径列表
2. **bo-5-distribute 负责插入图片**到 markdown 文件中
3. 使用 **Edit 工具**精确插入图片引用到合适位置
4. 图片引用格式：`![图片说明](img-{标题}/pxx-xxx.png)`
5. **必须在 frontmatter 中记录图片列表**（images字段）

**图片插入规则：**

小红书（3:4）：
```markdown
---

![封面图](img-{标题}/p1-cover.png)

# 标题

![核心观点](img-{标题}/p2-method.png)

正文内容...

![场景模板](img-{标题}/p3-scenarios.png)

更多内容...

![落地清单](img-{标题}/p4-checklist.png)

---

💬 你觉得AI调研靠谱吗？
👇 评论区聊聊你的看法~
```

公众号（4:3）：
```markdown
---

![封面图](img-{标题}/p1-cover.png)

# 标题

![核心观点](img-{标题}/p2-method.png)

正文内容...

![场景模板](img-{标题}/p3-scenarios.png)

更多内容...

![落地清单](img-{标题}/p4-checklist.png)

---

*关注我，了解更多AI趋势洞察*
```

**输出目录结构：**

```
05-图文工厂/
├── 小红书/
│   ├── img-{标题}/
│   │   ├── p1-cover.png
│   │   ├── p2-xxx.png
│   │   └── ...
│   └── {标题}-小红书.md   # 包含图片引用
├── 公众号/
│   ├── img-{标题}/
│   └── {标题}-公众号.md
└── ...
```

**注意事项：**
- bo-autoimages 会根据内容自动判断图片数量（3-6张）
- 图片命名规则：p1-cover（封面）, p2-pN（内容图）
- 如果 bo-autoimages 调用失败，记录错误但继续流程（不阻塞发布）

### 步骤4：平台特定改编

#### 4.1 小红书改编

**特点：**
- 视觉优先：需要配图建议
- 短小精悍：800-1000字
- 标题吸睛：使用emoji和关键词
- 标签重要：带上相关话题标签
- 金句密度：每段都要有金句

**改编策略：**
1. **标题优化：**
   ```
   原标题：AI孪生vs真人访谈：两种市场调研范式的较量
   小红书：🤖AI调研到底靠不靠谱？40万AI虚拟人冲击1500亿市场！

   标签：#AI #市场调研 #职场 #数据分析
   ```

2. **内容精简：**
   - 只保留最核心的2-3个观点
   - 每个观点配一个案例
   - 删除冗长的论证
   - 增加emoji增强可读性

3. **视觉元素：**
   > 注意：配图会自动生成并插入（步骤3.5），无需手动添加配图建议
   >
   > **重要**：bo-5-distribute 现在会自动调用 bo-autoimages 生成配图，并将图片引用插入到生成的 markdown 文件中。

4. **互动引导：**
   ```markdown
   💬 你觉得AI调研靠谱吗？
   👇 评论区聊聊你的看法~
   ```

#### 4.2 知乎改编

**特点：**
- 问答形式：从问题出发
- 专业深度：需要更多论证和数据
- 逻辑清晰：结构完整
- 引用规范：标注数据来源

**改编策略：**
1. **问题导入：**
   ```markdown
   问题：AI数字孪生和AI访谈，哪种市场调研方式更靠谱？

   背景：最近Simile（AI数字孪生）和Listen Labs（AI访谈）都获得了大额融资，看似相似，其实代表了两种截然不同的方法论...
   ```

2. **专业论证：**
   - 补充更多数据来源
   - 增加引用和注释
   - 展示专业深度
   - 提供决策框架

3. **结尾CTA：**
   ```markdown
   总结一下：
   - 没有最好的方法，只有最适合的方法
   - 根据你的场景选择合适的工具
   - AI是增强工具，不是替代工具

   觉得有帮助的话，点个赞同吧~
   ```

#### 4.3 微博改编

**特点：**
- 极短内容：300-500字
- 金句突出：一两个核心观点
- 互动性强：提问引导评论
- 话题标签：带上热搜话题

**改编策略：**
1. **核心观点：**
   ```markdown
   市场调研正在经历一场"范式之争"：

   一边是AI数字孪生（用AI模拟真人），一边是AI访谈（用AI访谈真人）。

   这不是技术路线的小分歧，而是对"什么是真实"的根本思考。
   ```

2. **案例支撑：**
   ```markdown
   CVS的案例很有说服力：
   - 用40万AI虚拟人测试营销话术
   - 准确率95%（但只能复现已知结论）
   - 所以他们永远不会停止与真实客户沟通

   这说明什么？AI是增强工具，不是替代工具。
   ```

3. **金句收尾：**
   ```markdown
   真正的敌人不是彼此，而是"不用AI调研"的保守心态。

   #AI #市场调研 #数字化转型
   ```

#### 4.4 抖音/视频号改编

**特点：**
- 脚本形式：分镜头描述
- 节奏快：60-90秒
- 视觉化：需要画面描述
- 黄金前3秒：抓住注意力

**改编策略：**
```markdown
## 抖音脚本：60秒

### 镜头1（0-3秒）- 黄金开头
**画面：** 分屏对比（左边AI虚拟人，右边真人访谈）
**文案：** "AI调研到底靠不靠谱？40万AI虚拟人正在冲击1500亿市场！"
**音效：** 紧张的背景音乐

### 镜头2（3-15秒）- 抛出问题
**画面：** 融资新闻截图
**文案：** "最近两家AI调研公司都拿到了大额融资，但他们的做法完全相反..."
**音效：** 音乐放缓

### 镜头3（15-45秒）- 核心内容
**画面：** 动画对比两种路径
**文案：** "一种用AI模拟真人，速度快成本低。一种用AI访谈真人，深度质量高。"
**字幕：** 关键词弹出

### 镜头4（45-55秒）- 结论
**画面：** CVS案例
**文案：** "但CVS说：AI测试准确率95%，但永远不会停止与真实客户沟通。"
**音效：** 强调音

### 镜头5（55-60秒）- CTA
**画面：** 主播出镜
**文案：** "AI是增强工具，不是替代工具。关注我，看懂AI时代的竞争力。"
**字幕：** 关注+点赞
```
