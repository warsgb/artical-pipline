# 灵感入库 Skill

这是内容生产工厂的第一阶段工作流：将你看到的灵感素材（文章、视频、金句、热点等）存入系统，并自动生成3个可写的选题。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"灵感入库"、"存进内容工厂"、"存灵感"
- 用户提供素材内容并要求生成选题
- 用户说"把这个存起来并生成3个选题"或类似表述
- 用户说"生成选题"并提供素材内容

## 工作流程

### 步骤0：识别素材类型

**检查用户输入：**

**场景A：用户提供了URL**
1. 使用 chrome-dev MCP 工具抓取网页内容：
   - 使用 `mcp__chrome-devtools__navigate_page` 导航到URL
   - 使用 `mcp__chrome-devtools__take_snapshot` 获取页面快照
   - 使用 WebFetch 或 defuddle 提取正文内容

2. 在 `00-网文收藏/` 目录下创建原始文章备份：
   - **文件位置：** `00-网文收藏/{{YYYY-MM-DD}}-{{文章标题}}.md`
   - **文件名格式：** 日期在前，如 `2026-03-08-文章标题.md`

3. **网文收藏文件模板：**
   ```yaml
   ---
   title: {{文章标题}}
   source_url: {{原始URL}}
   capture_date: {{DATE:YYYY-MM-DD}}
   capture_time: {{TIME:HH:mm}}
   tags: []
   ---

   # {{文章标题}}

   **来源：** {{网站/公众号名}}
   **发布时间：** {{如有}}
   **抓取时间：** {{DATE:YYYY-MM-DD HH:mm}}

   ---

   ## 原文内容

   {{完整正文内容}}

   ---

   *原文来源：{{URL}}*
   *本文件由 Claude + Obsidian 内容工厂自动抓取备份*
   ```

4. 提取文章核心内容，用于后续创建灵感卡
5. 在灵感卡的「关联笔记」中添加对网文收藏的引用

**场景B：用户提供了文本内容**
- 直接进入步骤1，不需要抓取网页

### 步骤1：创建灵感卡

**位置：** `01-灵感与素材库/1-日常灵感剪报/`

**文件名格式：** `YYYY-MM-DD-关键词.md`

使用 [[90-系统配置/模板/灵感模板]] 创建灵感卡，填写以下内容：

**YAML frontmatter：**
```yaml
---
title: {{标题}}
stage: 灵感
source: {{来源（如"Twitter"、"公众号"、"书籍"等）}}
capture_date: {{DATE:YYYY-MM-DD}}
tags: []
---
```

**内容部分：**
- **原始素材：** 粘贴灵感原文或摘要
- **来源链接：** 如果是网络内容，保留原始链接
- **核心亮点：** 提取2-4个��键点
- **我的联想：**
  - 可以延伸的话题
  - 可能的受众
  - 情绪共鸣点

### 步骤2：生成3个选题

**位置：** `02-选题池/待写选题库/`

**文件名格式：** `YYYY-MM-DD-选题标题.md`

每个选题独立文件，使用 [[90-系统配置/模板/选题模板]]，填写：

**YAML frontmatter：**
```yaml
---
title: {{选题标题}}
stage: 选题
content_type: {{类型（如"观点文章"、"教程"、"清单"等）}}
platform: {{目标平台（如"公众号"、"小红书"、"视频"等）}}
status: 待写
create_date: {{DATE:YYYY-MM-DD}}
priority: {{优先级（高/中/低）}}
---
```

**内容部分：**
- **核心观点：** 一句话概括这篇内容要表达什么
- **目标受众：**
  - 人群画像
  - 痛点/需求
  - 预期收获
- **平台适配：**
  - 主平台
  - 内容形式
  - 预估时长/字数
- **参考素材：** 链接回灵感卡
- **大纲草稿：** 3-5个要点

**选题要求：**
- 具体可写，不是泛泛而谈
- 有爆款潜力或实用价值
- 3个选题要有差异化角度（不同切入点、不同受众、不同形式）

### 步骤3：建立回链

**灵感卡中添加：**
```markdown
## 相关笔记
- [[00-网文收藏/YYYY-MM-DD-文章标题.md]]（如为URL来源）
- [[02-选题池/待写选题库/选题1文件名]]
- [[02-选题池/待写选题库/选题2文件名]]
- [[02-选题池/待写选题库/选题3文件名]]
```

**每个选题中添加：**
```markdown
## 关联笔记
- 灵感来源：[[01-灵感与素材库/1-日常灵感剪报/灵感卡文件名]]
```

## 输出要求

完成后，向用户报告：

1. **创建的文件路径**
   - 网文收藏（URL来源）：`00-网文收藏/YYYY-MM-DD-文章标题.md`（如有）
   - 灵感卡：`01-灵感与素材库/1-日常灵感剪报/YYYY-MM-DD-XXX.md`
   - 选题1：`02-选题池/待写选题库/YYYY-MM-DD-XXX.md`
   - 选题2：`02-选题池/待写选题库/YYYY-MM-DD-XXX.md`
   - 选题3：`02-选题池/待写选题库/YYYY-MM-DD-XXX.md`

2. **3个选题摘要**
   - 选题标题
   - 核心角度/差异化点

3. **确认信息**
   - "✅ 所有必要回链已建立"
   - "✅ 灵感已成功入库，可以进入下一阶段（大纲设计）"

## 使用示例

### 示例0：URL入库（自动抓取）

**用户输入：**
```
灵感入库：https://mp.weixin.qq.com/s/xxx (一篇关于AI创业的文章)
```

**执行结果：**
1. 抓取网页内容
2. 创建网文收藏：`00-网文收藏/2026-03-08-AI创业文章标题.md`
3. 创建灵感卡，提取核心亮点
4. 生成3个选题
5. 所有文件自动建立回链

**创建的文件：**
- ✅ 网文收藏：`00-网文收藏/2026-03-08-AI创业文章标题.md`
- ✅ 灵感卡：`01-灵感与素材库/1-日常灵感剪报/2026-03-08-XXX.md`
- ✅ 选题1-3

### 示例1：金句入库

**用户输入：**
```
把这条灵感存进内容工厂：

看到一篇文章说：「真正的效率不是做得更多，而是少做但必须做的事」
```

**执行结果：**
1. 创建灵感卡：`01-灵感与素材库/1-日常灵感剪报/2026-03-08-少即是多的效率观.md`
2. 创建3个选题：
   - `02-选题池/待写选题库/2026-03-08-为什么越忙的人效率越低.md`（反常识角度）
   - `02-选题池/待写选题库/2026-03-08-极简工作法的3个原则.md`（方法论角度）
   - `02-选题池/待写选题库/2026-03-08-从todo-list到not-to-do-list.md`（工具角度）
3. 所有文件自动建立回链

### 示例2：文章入库

**用户输入：**
```
灵感入库：
刚读完一篇关于AI写作的文章，提到三个关键点：
1. AI不是替代你，而是放大你的能力
2. 关键是学会prompt engineering
3. 要建立自己的知识库作为AI的上下文
```

**执行结果：**
1. 创建灵感卡，提取3个核心亮点
2. 生成3个不同角度的选题（如观点文章、实战教程、工具推荐）
3. 自动建立所有回链

## 注意事项

1. **文件命名：** 使用有意义的标题，避免过长的文件名
2. **日期格式：** 始终使用 YYYY-MM-DD 格式，日期放在文件名开头
3. **模板使用：** 必须使用指定的模板文件，保持结构一致
4. **回链完整性：** 确保灵感卡和所有选题之间都有双向链接
5. **选题质量：** 3个选题要有明显差异，避免重复或角度雷同
6. **YAML格式：** frontmatter必须格式正确，字段完整
7. **URL处理：** 如果用户输入URL，先抓取内容保存到 `00-网文收藏/`，再继续后续流程
8. **网文收藏：** 文件名格式为 `YYYY-MM-DD-文章标题.md`，日期在前
