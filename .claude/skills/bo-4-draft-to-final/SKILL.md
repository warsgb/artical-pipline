# 初稿转终稿 Skill

这是内容生产工厂的第四阶段工作流：将已完成的初稿进行润色、精简、优化，转化为可以发布的终稿文档。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"初稿转终稿"、"生成终稿"、"创建终稿"
- 说"润色终稿"、"优化文章"、"精简内容"
- 说"/生成终稿"、"/润色"
- 对一个已通过的初稿文件说"推进到终稿"

**前置条件：**
- 初稿文件的 status 应该是"已通过"或"待完善"
- 如果 status 是"写作中"，会提示用户先完成初稿

## 工作流程

### 步骤1：读取初稿文件

**操作：**
1. 读取用户指定的初稿文件
2. 提取初稿的关键信息：
   - `title` - 标题
   - `content_type` - 内容类型
   - `platform` - 目标平台
   - `source_outline` - 来源大纲
   - `word_count` - 当前字数
   - 初稿内容

### 步骤2：分析初稿质量

对初稿进行全面分析：

1. **结构分析：**
   - 开篇是否吸引人
   - 正文逻辑是否清晰
   - 结尾是否有力
   - 模块之间是否连贯

2. **内容分析：**
   - 观点是否明确
   - 案例是否充分
   - 数据是否准确
   - 论证是否有力

3. **语言分析：**
   - 表达是否流畅
   - 是否有冗余
   - 是否有错别字
   - 语言风格是否统一

4. **优化建议：**
   - 哪些部分需要精简
   - 哪些部分需要补充
   - 哪些部分需要重组
   - 建议的最终字数

### 步骤3：执行润色优化

**优化维度：**

#### 3.1 结构优化

- **开篇优化：**
  - 去除冗余的背景介绍
  - 强化钩子的冲击力
  - 更快进入主题
  - 字数控制在300-500字

- **正文优化：**
  - 合并重复的观点
  - 删除不必要的论证
  - 调整模块顺序
  - 强化模块之间的过渡

- **结尾优化：**
  - 强化金句
  - 明确行动号召
  - 删除拖沓的总结
  - 字数控制在200-300字

#### 3.2 内容优化

- **案例优化：**
  - 选择最有代表性的案例
  - 删除次要案例
  - 深化主要案例的描述
  - 确保案例支撑观点

- **数据优化：**
  - 核实数据来源
  - 删除不重要的数据
  - 用数据强化关键观点
  - 数据可视化建议

- **观点优化：**
  - 每个观点一句话说清楚
  - 删除模糊不清的表达
  - 强化核心论点
  - 确保观点有支撑

#### 3.3 语言优化

- **精简冗余：**
  - 删除重复的表达
  - 去除修饰性词语
  - 精简长句为短句
  - 删除不必要的连接词

- **增强表达：**
  - 使用更精准的词汇
  - 增加金句密度
  - 优化节奏感
  - 提升可读性

- **统一风格：**
  - 保持语言风格一致
  - 统一称呼和术语
  - 调整语气
  - 确保符合目标受众

#### 3.4 视觉优化

- **标题层级：**
  - 优化小标题
  - 确保层级清晰
  - 增加吸引力

- **格式优化：**
  - 使用列表增强可读性
  - 使用引用突出重点
  - 使用表格展示对比
  - 适当使用加粗

### 步骤4：创建终稿文件

**文件位置：** `03-内容工厂/3-终稿发布区/YYYY-MM-DD-{{title}}-终稿.md`

**使用模板：** `[[90-系统配置/模板/文章模板]]`

**填充字段：**
```yaml
---
title: "{{初稿title}}"
stage: 终稿
content_type: "{{初稿content_type}}"
platform: "{{初稿platform}}"
status: 终稿待发布
source_outline: "[[03-内容工厂/2-初稿打磨区/原初稿文件]]"
create_date: "{{DATE:YYYY-MM-DD}}"
update_date: "{{DATE:YYYY-MM-DD}}"
word_count: "{{最终字数}}字"
advance_note: "从初稿润色而来，已优化结构和语言"
---
```

### 步骤5：生成终稿内容

**基于初稿，生成优化后的终稿：**

#### 5.1 优化开篇

**原则：** 更快进入主题，更有冲击力

```
{{精简后的钩子，300-500字}}

{{删除冗余背景}}

{{直接抛出问题或观点}}
```

#### 5.2 优化正文

**原则：** 精简30%，强化核心

```
### {{优化后的小标题}}

{{核心观点，一句话说清楚}}

{{保留最有力的案例}}

{{删除次要论证}}

{{用数据和金句强化}}
```

#### 5.3 优化结尾

**原则：** 有力收尾，明确行动

```
## 结语

{{2-3句话总结}}

{{金句，加粗}}

{{明确的行动号召}}

{{删除拖沓的总结}}
```

### 步骤6：生成修改说明

**在终稿末尾添加：**
```markdown
---

## 修改说明

### 主要改动
- 结构调整：{{哪些部分调整了顺序}}
- 内容精简：从{{初稿字数}}字精简到{{终稿字数}}字（减少{{X}}%）
- 语言优化：{{哪些表达优化了}}

### 优化重点
1. {{优化点1}}
2. {{优化点2}}
3. {{优化点3}}

### 保留精华
- {{保留的案例1}}
- {{保留的金句1}}

### 发布建议
- 推荐配图：{{配图建议}}
- 推荐发布时间：{{时间建议}}
- 预期阅读时间：{{分钟}}分钟
```

### 步骤7：更新初稿状态

**更新初稿文件的YAML：**
```yaml
---
title: "{{原有title}}"
stage: 初稿
status: 已通过  # 从"写作中/待完善"更新为"已通过"
update_date: "{{DATE:YYYY-MM-DD}}"
advance_note: "已生成终稿：[[03-内容工厂/3-终稿发布区/YYYY-MM-DD-XXX-终稿.md]]"
---
```

**在初稿的「关联笔记」添加：**
```markdown
## 关联笔记
- 大纲来源：[[...]]
- 已生成终稿：[[03-内容工厂/3-终稿发布区/YYYY-MM-DD-XXX-终稿.md]]
```

## 润色原则

1. **精简优先：** 能用10个字说清楚的，不用20个字
2. **案例聚焦：** 保留最有代表性的1-2个案例，删除次要案例
3. **数据精准：** 只用关键数据，删除不重要的数据
4. **金句强化：** 在关键位置使用金句
5. **节奏优化：** 长短句结合，提升阅读节奏
6. **口语化：** 避免过于书面化，让读者更容易理解
7. **视觉化：** 用表格、列表、引用块等增强可读性
8. **统一风格：** 保持语言风格、称呼、术语的一致性

## 注意事项

1. **初稿完整性：** 确保初稿已经完成再生成终稿
2. **精简适度：** 不要过度精简导致内容缺失
3. **保留精华：** 初稿中的优秀部分应该保留
4. **提升质量：** 终稿应该在初稿基础上明显提升
5. **保持原意：** 优化表达但不改变原意
6. **符合平台：** 根据发布平台调整风格
7. **回链建立：** 务必在初稿中添加回链到终稿
8. **修改说明：** 提供清晰的修改说明，方便用户了解改动
9. **可发布性：** 终稿应该达到"可直接发布"的标准
10. **模板使用：** 必须使用指定的文章模板
