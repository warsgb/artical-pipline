# 状态查询 Skill

这是内容生产工厂的状态查询工具：快速查看任何内容的当前状态、上下游关系、完整生产链条。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"/状态"、"查看状态"、"查询状态"
- 说"这个内容进行到哪了"、"进度如何"
- 说"查看内容链"、"查看关系链"
- 说"这个选题的状态"、"这个大纲的状态"
- 对任何文档说"状态"

## 工作流程

### 步骤1：读取当前文档状态

**操作：**
1. 读取用户指定的文件（如果未指定，读取当前打开的文件）
2. 提取 YAML frontmatter
3. 解析关键字段：`stage`、`status`、`title`、`create_date`、`update_date`

**解析示例：**
```yaml
title: "AI写作方法论"
stage: 大纲
status: 待完善
create_date: "2026-03-01"
update_date: "2026-03-05"
source_topic: "[[02-选题池/2-已立项/2026-03-01-AI写作方法论.md]]"
```

### 步骤2：追溯上游关系

**目标：** 找到内容的来源和前置文档

**追溯逻辑：**
1. 如果有 `source_topic` → 找到选题文件
2. 读取选题的「关联笔记」→ 找到灵感文件（如有）
3. 如果有 `source_outline` → 找到上游大纲/初稿

**示例追溯：**
```
当前：大纲 (AI写作方法论)
  ↑ source_topic
选题：02-选题池/2-已立项/2026-03-01-AI写作方法论.md
  ↑ 关联笔记-灵感来源
灵感：01-灵感与素材库/1-日常灵感剪报/2026-02-28-XXX.md
```

### 步骤3：追溯下游关系

**目标：** 找到基于此内容创建的下游文档

**追溯逻辑：**
1. 搜索 vault 中所有引用当前文件的文档
2. 通过 `source_topic` 或 `source_outline` 字段匹配
3. 读取「关联笔记」部分，查找下游链接

**示例追溯：**
```
当前：大纲 (AI写作方法论)
  ↓ 被引用
初稿：03-内容工厂/2-初稿打磨区/2026-03-05-AI写作方法论-初稿.md
  ↓ source_outline
  ↓ 被引用
终稿：03-内容工厂/3-终稿发布区/2026-03-08-AI写作方法论-终稿.md
```

### 步骤4：组装完整状态面板

**输出格式：**

```markdown
## 📊 内容状态面板

### 基本信息
- **标题：** {{title}}
- **当前阶段：** {{stage}}
- **当前状态：** {{status}}
- **创建时间：** {{create_date}}
- **更新时间：** {{update_date}}
- **推进备注：** {{advance_note 或 "无"}}

### 状态进度条
{{根据阶段绘制进度条}}

灵感 → 选题 → 大纲 → 初稿 → 终稿 → 发布 → 归档
 ✅    ✅    🔄    ⏸️   ⏸️   ⏸️   ⏸️

图例：✅ 已完成 | 🔄 当前阶段 | ⏸️ 待开始

### 完整内容链

#### 上游（来源）
1. **灵感** - [[灵感文件路径]] - ✅ 已转化
   - 创建时间：{{灵感create_date}}

2. **选题** - [[选题文件路径]] - ✅ 已通过
   - 核心观点：{{选题核心观点}}
   - 创建时间：{{选题create_date}}

#### 当前阶段
3. **大纲** - [[当前文件]] - 🔄 待完善
   - 结构要点：{{大纲结构}}
   - 最后更新：{{update_date}}

#### 下游（产出）
⏸️ **初稿** - 待创建
⏸️ **终稿** - 待创建
⏸️ **发布** - 待发布
⏸️ **归档** - 待归档

### 可执行操作
- [ ] 继续完善大纲
- [ ] 完成后使用 /推进 创建初稿
- [ ] 查看 /状态 了解进度
```

## 输出要求

1. **状态面板标题**
   ```
   📊 「{{title}}」状态面板
   ```

2. **关键信息优先**
   - 当前阶段和状态放在最前面
   - 用颜色/符号突出显示当前状态

3. **可执行性**
   - 明确列出下一步可以做什么
   - 提供可点击的wikilinks

4. **可视化**
   - 使用进度条或流程图展示内容链
   - 用符号（✅🔄⏸️）表示各阶段状态

5. **信息完整**
   - 展示完整的上下游关系
   - 显示关键时间节点
   - 提供跳转链接

## 注意事项

1. **文件可读性**：如果上游文件被移动或删除，提示用户文件不存在
2. **状态准确性**：严格按照YAML字段显示状态，不要臆测
3. **链接有效性**：确保所有wikilinks可以正确跳转
4. **信息层次**：先显示关键信息，再显示详细关系链
5. **可操作性**：明确告诉用户下一步可以做什么
6. **符号一致性**：统一使用 ✅（已完成）、🔄（进行中）、⏸️（待开始）
7. **时间显示**：显示创建时间、更新时间等关键时间点
8. **多语言支持**：如果用户用英文触发，也可以用英文输出
9. **错误处理**：如果文件缺少必要字段，给出友好提示
10. **性能考虑**：避免读取过多文件，只读取必要的上下游关系
