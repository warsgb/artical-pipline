# 大纲转初稿 Skill

这是内容生产工厂的第三阶段工作流：将已通过的大纲转化为完整的初稿文档，填充内容并形成可读的文章。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"大纲转初稿"、"生成初稿"、"创建初稿"
- 说"开始写初稿"、"按大纲写文章"
- 说"/生成初稿"、"/写初稿"
- 对一个已通过的大纲文件说"推进到初稿"

**前置条件：**
- 大纲文件的 status 应该是"已通过"或"待完善"
- 如果 status 是"待完善"，会提示用户先完善大纲

## 工作流程

### 步骤1：读取大纲文件

**操作：**
1. 读取用户指定的大纲文件
2. 提取大纲的关键信息：
   - `title` - 标题
   - `content_type` - 内容类型
   - `platform` - 目标平台
   - `source_topic` - 来源选题
   - 大纲结构（开篇、正文模块、结尾）

**示例大纲信息：**
```markdown
# 内容大纲

## 开篇（钩子）
- 钩子类型：反常识
- 具体文案：想象这样一个画面...

## 正文结构
### 模块1：背景/问题
- 要点：...
- 案例/数据：...

### 模块2：核心观点
...
```

### 步骤2：分析大纲结构

分析大纲的完整度和写作方向：

1. **开篇钩子：**
   - 类型：痛点/好奇/共鸣/反常识
   - 字数：300-500字
   - 目标：吸引读者继续阅读

2. **正文模块：**
   - 模块数量：通常3-5个
   - 逻辑关系：并列/递进/对比
   - 每个模块的字数分配

3. **结尾CTA：**
   - 金句收尾
   - 行动号召
   - 互动引导

4. **预估总字数：**
   - 公众号：2000-3000字
   - 小红书：800-1000字
   - 知乎：1500-2500字

### 步骤3：创建初稿文件

**文件位置：** `03-内容工厂/2-初稿打磨区/YYYY-MM-DD-{{title}}-初稿.md`

**使用模板：** `[[90-系统配置/模板/文章模板]]`

**填充字段：**
```yaml
---
title: "{{大纲title}}"
stage: 初稿
content_type: "{{大纲content_type}}"
platform: "{{大纲platform}}"
status: 写作中
source_outline: "[[03-内容工厂/1-大纲挑选区/原大纲文件]]"
create_date: "{{DATE:YYYY-MM-DD}}"
update_date: "{{DATE:YYYY-MM-DD}}"
word_count: "预计{{字数}}字"
advance_note: "从大纲转化而来，正在写作中"
---
```

### 步骤4：生成初稿内容

**根据大纲结构，按模块生成内容：**

#### 4.1 生成开篇

**目标：** 300-500字，吸引读者

**结构：**
```markdown
{{钩子类型：反常识/痛点/好奇/共鸣}}

想象一下：{{场景描述}}

这不是{{否定句}}，这是{{肯定句}}。

{{问题抛出：当AI能做的事越来越多，人的价值在哪里？}}

{{过渡：这篇文章给你答案}}
```

#### 4.2 生成正文模块

**每个模块的结构：**
```markdown
### {{模块标题}}

{{引言句：2-3句话引入本模块}}

{{核心观点：1句话总结}}

{{论证：}}
- {{论点1 + 案例/数据}}
- {{论点2 + 案例/数据}}
- {{论点3 + 案例/数据}}

{{小结：1-2句话总结本模块要点}}

---
```

**注意：**
- 每个模块500-800字
- 使用案例和数据支撑观点
- 保持逻辑连贯
- 适当使用小标题分割

#### 4.3 生成结尾

**目标：** 200-300字，升华+行动号召

**结构：**
```markdown
## 结语

{{总结：2-3句话回顾全文}}

{{金句：**核心观点，加粗显示**}}

{{行动号召：}}
从今天开始，{{具体行动建议}}。

{{预期效果：}}
一周后，{{效果1}}。
一个月后，{{效果2}}。
一年后，{{效果3}}。

{{祝福语：}}
祝你在{{领域}}，成为{{目标角色}}。

---
```

### 步骤5：补充案例和数据

**在初稿中标注需要补充的位置：**
```markdown
<!-- TODO: 补充具体案例 -->
<!-- TODO: 补充数据支持 -->
<!-- TODO: 查证来源 -->
```

### 步骤6：更新大纲状态

**更新大纲文件的YAML：**
```yaml
---
title: "{{原有title}}"
stage: 大纲
status: 已通过  # 从"待完善"更新为"已通过"
update_date: "{{DATE:YYYY-MM-DD}}"
advance_note: "已生成初稿：[[03-内容工厂/2-初稿打磨区/YYYY-MM-DD-XXX-初稿.md]]"
---
```

**在大纲的「关联笔记」添加：**
```markdown
## 关联笔记
- 选题来源：[[...]]
- 已生成初稿：[[03-内容工厂/2-初稿打磨区/YYYY-MM-DD-XXX-初稿.md]]
```

## 输出要求

完成后，向用户报告：

1. **创建成功信息**
   ```
   ✅ 已从大纲「{{title}}」生成初稿

   📄 初稿文件：03-内容工厂/2-初稿打磨区/YYYY-MM-DD-XXX-初稿.md
   - Stage: 初稿
   - Status: 写作中
   - 预估字数：{{字数}}字
   - 已建立回链到原大纲
   ```

2. **初稿结构概览**
   ```
   📝 初稿内容：
   - 开篇钩子：{{字数}}字
   - 正文模块：{{X个模块}}
   - 结尾CTA：{{字数}}字
   - 总字数：{{总字数}}字

   ⚠️ 待补充：
   - {{N个案例需要补充}}
   - {{N个数据需要查证}}
   ```

3. **下一步提示**
   ```
   📌 下一步：
   - 查看初稿，补充案例和数据
   - 润色语言，优化表达
   - 完成后使用 /推进 或使用 bo-draft-to-final 生成终稿
   ```

## 写作原则

1. **按大纲写作：** 严格按照大纲的结构写作，不随意增删模块
2. **案例先行：** 能用案例说明的，就不用抽象论述
3. **数据支撑：** 关键观点必须有数据支撑
4. **金句点缀：** 在关键位置使用金句，增强可传播性
5. **口语化：** 避免过于学术化，让普通读者也能看懂
6. **短段落：** 每段3-5句话，不超过150字
7. **小标题：** 使用小标题分割长内容，提升可读性
8. **视觉化：** 适当使用表格、列表、引用块等格式

## 注意事项

1. **大纲完整性：** 确保大纲已经相对完善再生成初稿
2. **内容质量：** 初稿应该达到"可读"的标准，而不是只有框架
3. **案例标注：** 对于不确定的案例或数据，用TODO标注
4. **字数控制：** 根据平台特点控制字数，不要过多或过少
5. **语言风格：** 根据目标受众调整语言风格
6. **逻辑连贯：** 确保各个模块之间的逻辑连贯
7. **回链建立：** 务必在大纲中添加回链到初稿
8. **模板使用：** 必须使用指定的文章模板
9. **可修改性：** 初稿是待完善的，鼓励用户后续修改
10. **平台适配：** 根据不同平台调整写作风格和结构
