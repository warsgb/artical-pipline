# 选题转大纲 Skill

这是内容生产工厂的第二阶段工作流：将已通过的选题转化为详细的大纲文档，为后续写作提供清晰的框架。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"选题转大纲"、"生成大纲"、"创建大纲"
- 说"开始写大纲"、"给这个选题写大纲"
- 说"/生成大纲"、"/转大纲"
- 对一个已通过的选题文件说"推进到大纲"

**前置条件：**
- 选题文件的 status 应该是"已通过"或"进行中"
- 如果 status 是"待写"，会先更新为"进行中"或"已通过"

## 工作流程

### 步骤1：读取选题文件

**操作：**
1. 读取用户指定的选题文件
2. 提取选题的关键信息：
   - `title` - 标题
   - `content_type` - 内容类型
   - `platform` - 目标平台
   - `核心观点` - 核心观点
   - `目标受众` - 目标受众
   - `大纲草稿` - 如果有

**示例选题信息：**
```yaml
title: "AI孪生 vs 真人访谈：两种市场调研范式的较量"
content_type: 对比分析
platform: 公众号
核心观点: 市场调研正在经历一场"范式之争"
目标受众: 市场调研从业者、AI产品经理
大纲草稿: 11个章节的草稿结构
```

### 步骤2：分析选题结构

根据选题文件中的信息，分析：

1. **文章类型：**
   - 观点文章：需要论证结构
   - 教程类：需要步骤结构
   - 对比分析：需要对比结构
   - 清单体：需要并列结构

2. **平台特点：**
   - 公众号：深度+逻辑，2000-3000字
   - 小红书：视觉+金句，800-1000字
   - 知乎：问答+论证，1500-2500字

3. **目标受众：**
   - 专业人士：需要深度和案例
   - 普通读者：需要通俗和实用
   - 混合受众：需要兼顾深度和可读性

### 步骤3：创建大纲文件

**文件位置：** `03-内容工厂/1-大纲对焦区/YYYY-MM-DD-{{title}}.md`

**使用模板：** `[[90-系统配置/模板/大纲模板]]`

**填充字段：**
```yaml
---
title: "{{选题title}}"
stage: 大纲
content_type: "{{选题content_type}}"
platform: "{{选题platform}}"
status: 待完善
source_topic: "[[02-选题池/待写选题库/原选题文件]]"
create_date: "{{DATE:YYYY-MM-DD}}"
update_date: "{{DATE:YYYY-MM-DD}}"
advance_note: "从选题转化而来"
---
```

### 步骤4：生成大纲内容

根据选题的「大纲草稿」或「核心观点」，生成完整的大纲结构：

```markdown
# 内容大纲

## 开篇（钩子）
- 钩子类型：痛点/好奇/共鸣/反常识
- 具体文案：
  - {{根据选题的核钩子类型生成}}
  - {{用300-500字吸引读者}}

## 正文结构

### 模块1：背景/问题
- 要点：
  - {{要点1}}
  - {{要点2}}
- 案例/数据：
  - {{案例1}}
  - {{数据支持}}

### 模块2：核心观点1
- 要点：
  - {{要点1}}
  - {{要点2}}
- 案例/数据：
  - {{案例}}

### 模块3：核心观点2
- 要点：
  - {{要点1}}
  - {{要点2}}
- 案例/数据：
  - {{案例}}

### 模块4：实操/行动建议
- 步骤1：
- 步骤2：
- 步骤3：

## 结尾（升华/CTA）
- 金句收尾：
- 行动号召：
- 互动引导：

## 平台适配
- 小红书：封面标题、标签
- 公众号：配图建议、排版要点
- 抖音/视频：镜头脚本、时长分配
```

### 步骤5：更新选题状态

**更新选题文件的YAML：**
```yaml
---
title: "{{原有title}}"
stage: 选题
status: 已通过  # 从"待写/进行中"更新为"已通过"
update_date: "{{DATE:YYYY-MM-DD}}"
advance_note: "已生成大纲：[[03-内容工厂/1-大纲对焦区/YYYY-MM-DD-XXX.md]]"
# ... 其他字段保持不变
---
```

**在选题的「关联笔记」添加：**
```markdown
## 关联笔记
- 灵感来源：[[...]]
- 已生成大纲：[[03-内容工厂/1-大纲对焦区/YYYY-MM-DD-XXX.md]]
```

## 输出要求

完成后，向用户报告：

1. **创建成功信息**
   ```
   ✅ 已从选题「{{title}}」生成大纲

   📄 大纲文件：03-内容工厂/1-大纲对焦区/YYYY-MM-DD-XXX.md
   - Stage: 大纲
   - Status: 待完善
   - 已建立回链到原选题
   ```

2. **大纲结构概览**
   ```
   📋 大纲结构：
   - 开篇钩子：{{钩子类型}}
   - 正文模块：{{X个模块}}
   - 结尾CTA：{{行动号召}}
   总字数预估：{{字数}}字
   ```

3. **下一步提示**
   ```
   📌 下一步：
   - 查看大纲文件，完善细节
   - 补充案例和数据支持
   - 完成后使用 /推进 或使用 bo-outline-to-draft 生成初稿
   ```

## 注意事项

1. **选题状态检查**：确保选题已经相对完善，才生成大纲
2. **大纲完整性**：生成的 outline 应该包含开篇、正文、结尾三个部分
3. **平台适配**：根据不同平台调整大纲的侧重点
4. **案例预留**：在大纲中预留案例和数据的填充位置
5. **字数预估**：根据大纲结构预估最终字数
6. **回链建立**：务必在选题中添加回链到大纲
7. **文件命名**：使用日期+标题的命名格式
8. **灵活性**：如果选题已有详细大纲草稿，直接使用；如果没有，根据核心观点生成
9. **可扩展性**：大纲应该易于后续扩展，而不是过于死板
10. **模板使用**：必须使用指定的大纲模板，保持结构一致
