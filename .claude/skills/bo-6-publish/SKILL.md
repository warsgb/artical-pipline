# 发布内容 Skill

这是内容生产工厂的第6阶段：将已创建的多平台版本标记为已发布，记录发布信息。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"/发布"、"发布内容"、"标记为已发布"
- 说"发布到平台名"、"发布到小红书"
- 对一个小红书/抖音/知乎文件说"/发布"
- 说"这条内容已发布"

**前置条件：**
- 目标文件应该是通过 bo-5-distribute 创建的多平台版本
- 用户应指定发布平台

## 工作流程

### 步骤1：读取目标文件

**操作：**
1. 读取用户指定的多平台版本文件
2. 提取关键信息：
   - `title` - 标题
   - `platform` - 目标平台
   - `content_type` - 内容类型

### 步骤2：确认发布信息

**询问用户（如果未提供）：**
1. **发布平台：** 小红书/抖音/知乎/微博/其他
2. **发布时间：** 立即发布/预约发布（日期时间）
3. **发布链接：** 如果已经发布，提供链接

### 步骤3：更新文件状态

**更新文件的YAML：**
```yaml
---
title: "{{原有title}}"
stage: 已发布
status: 已发布
publish_date: "{{DATE:YYYY-MM-DD}}"
publish_time: "{{TIME:HH:mm}}"
publish_platform: "{{平台名称}}"
publish_url: "{{发布链接（如有）}}"
update_date: "{{DATE:YYYY-MM-DD}}"
advance_note: "已发布到{{平台名称}}"
---
```

### 步骤4：添加发布信息

**在文件末尾添加：**
```markdown
---

## 发布信息

- **发布平台：** {{平台名称}}
- **发布时间：** {{YYYY-MM-DD HH:mm}}
- **发布链接：** {{链接（如有）}}
- **阅读量/播放量：** {{待更新}}
- **点赞数：** {{待更新}}
- **评论数：** {{待更新}}
- **转发数：** {{待更新}}

---

*发布时间：{{YYYY-MM-DD HH:mm}}*
```

## 输出要求

完成后，向用户报告：

1. **发布成功信息**
   ```
   ✅ 已将「{{title}}」标记为已发布

   📝 发布信息：
   - 发布平台：{{平台名称}}
   - 发布时间：{{YYYY-MM-DD HH:mm}}
   - 发布链接：{{链接（如有）}}
   ```

2. **下一步提示**
   ```
   📌 下一步：
   - 可以继续发布到其他平台
   - 或使用 /归档 归档整个内容链
   ```

## 使用示例

### 示例1：发布小红书版本

**用户输入：**
```
/发布 小红书
```
（小红书版本已创建好）

**执行结果：**
```
✅ 已将「AI Agent工具入门指南」小红书版本标记为已发布

📝 发布信息：
- 发布平台：小红书
- 发布时间：2026-03-08 18:00
- 标签：#AI #工具 #入门

📌 下一步：
- 可以继续发布到知乎、抖音等平台
- 或使用 /归档 归档整个内容链
```

### 示例2：发布抖音脚本

**用户输入：**
```
/发布 抖音
```

**执行结果：**
```
✅ 已将「AI Agent工具入门指南」抖音脚本标记为已发布

📝 发布信息：
- 发布平台：抖音
- 发布时间：2026-03-08

📌 下一步：
- 可以开始制作视频
- 完成后可以使用 /归档 归档
```

## 注意事项

1. **文件确认：** 确保目标文件是已创建的多平台版本
2. **状态更新：** 更新 status 为"已发布"
3. **记录时间：** 准确记录发布时间
4. **链接收集：** 尽可能收集发布链接
5. **数据追踪：** 提示用户后续更新数据
