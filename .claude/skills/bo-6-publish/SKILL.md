# 发布内容 Skill

这是内容生产工厂的第6阶段：将已创建的多平台版本真实发布到各个平台。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"/发布"、"发布内容"
- 说"发布到平台名"、"发布到小红书"
- 对一个平台版本文件说"/发布"
- 说"这条内容已发布"

**前置条件：**
- 目标文件应该是通过 bo-5-distribute 创建的多平台版本
- 文件应位于 `05-图文工厂/{平台名}/` 目录下

## 工作流程

### 步骤1：识别发布平台

**从文件路径或用户输入中识别平台：**

| 平台 | 文件路径关键词 | 别名 |
|------|--------------|------|
| 微信公众号 | `公众号/`、`wechat/`、`wx/` | 公众号、微信 |
| 小红书 | `小红书/`、`xhs/` | RED、小红书 |
| 知乎 | `知乎/`、`zhihu/` | 知乎 |
| 微博 | `微博/`、`weibo/` | 微博 |
| 抖音 | `抖音/`、`douyin/` | 抖音 |

**优先级：**
1. 从文件路径自动识别（如 `05-图文工厂/公众号/文章.md`）
2. 用户明确指定（如 `/发布 公众号`）
3. 从文件的 `platform` 字段读取

### 步骤2：检查文件前置条件

**验证以下信息：**

```yaml
# 检查清单
✅ 文件存在性
✅ 文件内容完整性
✅ 配图是否已生成（对于公众号和小红书）
✅ 格式是否符合平台要求
```

**对于公众号和小红书：**
- 检查 `05-图文工厂/{平台}/img-{标题}/` 目录是否存在
- 验证图片文件数量（建议3-6张）
- 如配图缺失，提示用户先调用 `/bo-autoimages` 生成配图

**配图检查示例：**
```bash
# 公众号配图检查
图片目录：05-图文工厂/公众号/img-{标题}/
预期图片：p1-cover.png, p2-xxx.png, p3-xxx.png...
```

### 步骤3：调用对应平台的发布技能

根据识别的平台，调用相应的发布子技能：

#### 3.1 微信公众号发布 ✅

**触发条件：** 平台 = "公众号" 或 "微信"

**执行流程：**
```bash
# 调用 bo-x-pubwechat
/bo-x-pubwechat @公众号版本文件路径
```

**bo-x-pubwechat 会自动：**
1. 读取 Markdown 文件
2. 提取元数据（标题、作者、摘要）
3. 转换为 HTML（包含 base64 图片，如果图片存在）
4. 启动 Chrome 浏览器
5. 登录微信公众号（如未登录）
6. 填充标题、内容、摘要
7. 保存到草稿箱
8. 保持浏览器窗口打开供用户确认发布

**返回信息：**
- ✅ 发布成功：草稿已保存
- ⚠️ 需要用户手动：点击"发布"按钮正式发布

**完成后获取：**
- 草稿箱链接
- 发布状态

### 步骤4：更新文件状态

**真实发布完成后（或草稿保存后），更新文件的 YAML：**

```yaml
---
title: "{{原有title}}"
stage: 已发布
status: 已发布
publish_date: "{{DATE:YYYY-MM-DD}}"
publish_time: "{{TIME:HH:mm}}"
publish_platform: "{{平台名称}}"
publish_url: "{{发布链接或草稿箱链接}}"
publish_status: "{{状态：已发布/草稿箱}}"
update_date: "{{DATE:YYYY-MM-DD}}"
advance_note: "已发布到{{平台名称}}"
---
```

**在文件末尾添加发布信息：**
```markdown
---

## 发布信息

- **发布平台：** {{平台名称}}
- **发布时间：** {{YYYY-MM-DD HH:mm}}
- **发布方式：** Chrome CDP 自动化
- **发布链接：** {{链接（如有）}}
- **草稿箱链接：** {{草稿箱链接（微信公众号）}}

**数据追踪（待更新）：**
- 阅读量/播放量：
- 点赞数：
- 评论数：
- 转发数：

---

*发布时间：{{YYYY-MM-DD HH:mm}}*
```

## 错误处理

### 错误1：配图未生成

**场景：** 发布公众号/小红书，但配图目录不存在

**处理：**
```
⚠️  检测到配图未生成

建议：
1. 先运行 /bo-autoimages 生成配图
2. 或选择不使用配图继续发布

是否先生成配图？(y/n)
```

**用户选择 y：**
```bash
调用 /bo-autoimages @公众号版本文件
然后继续发布流程
```

### 错误2：文件路径错误

**检查：**
- 文件是否在 `05-图文工厂/{平台}/` 目录下
- 文件命名是否正确

**修复提示：**
```
❌ 文件路径不正确

期望路径：05-图文工厂/{平台}/YYYY-MM-DD-{{title}}-{{platform}}.md
实际路径：{{实际路径}}

请检查文件位置后重试
```

### 错误3：平台不支持

```
❌ 暂不支持该平台的自动发布

平台：{{平台名}}

当前支持的平台：
✅ 公众号（自动化发布到草稿箱）
⏳ 小红书（待实现）
⏳ 知乎（待实现）
⏳ 微博（待实现）
⏳ 抖音（待实现）

建议：手动发布后使用 /发布 命令记录状态
```

## 注意事项

### 通用注意事项
1. **文件位置：** 确保目标文件在正确的平台目录下
2. **状态更新：** 发布后及时更新文件状态
3. **数据追踪：** 发布后定期更新数据指标
4. **链接收集：** 尽可能收集发布链接用于追踪

### 微信公众号特定
5. **草稿状态：** 保存到草稿箱后需要手动点击"发布"
6. **浏览器窗口：** 发布后浏览器窗口保持打开
7. **配图检查：** 确保图片已生成（应在 `05-图文工厂/公众号/img-{标题}/` 目录）

### 小红书特定
8. **手动发布：** 当前需手动复制粘贴
9. **配图上传：** 逐张上传图片到小红书
10. **标签使用：** 使用话题标签提升曝光

### 其他平台
11. **知乎：** 需要手动发布
12. **微博：** 需要手动发布
13. **抖音：** 需要录制视频后上传
