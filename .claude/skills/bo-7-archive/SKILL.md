# 内容归档 Skill

这是内容生产工厂的最后阶段工作流：将已发布的内容及其整个生产链条（灵感→选题→大纲→初稿→终稿）归档到专门区域，创建索引文件和复盘卡片。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"/归档"、"归档这个内容"
- 对已发布的终稿说"归档"
- 说"内容已完成，归档"
- 说"这个系列可以归档了"

**前置条件：**
- 归档目标必须是 **已发布** 的内容（status=已发布）
- 如果内容未发布，提示用户先完成发布

## 工作流程

### 步骤1：识别内容链

**目标：** 找到该内容的完整生产链条

**操作：**
1. 从用户指定的终稿文件开始
2. 读取 `source_outline` 字段，找到初稿
3. 读取初稿的 `source_outline`，找到大纲
4. 读取大纲的 `source_topic`，找到选题
5. 读取选题的「关联笔记」，找到灵感（如有）

**追溯示例：**
```
终稿：03-内容工厂/3-终稿发布区/2026-03-08-XXX-终稿.md
  ↓ source_outline
初稿：03-内容工厂/2-初稿打磨区/2026-03-08-XXX-初稿.md
  ↓ source_outline
大纲：03-内容工厂/1-大纲挑选区/2026-03-08-XXX.md
  ↓ source_topic
选题：02-选题池/2-已立项/2026-03-08-XXX.md
  ↓ 关联笔记
灵感：01-灵感与素材库/1-日常灵感剪报/YYYY-MM-DD-XXX.md（可选）
```

### 步骤2：创建归档目录

**目录结构：**
```
06-已发布归档/
├── YYYY-MM/                          # 按年月分组
│   └── {{内容标题}}/
│       ├── {{内容标题}}-灵感.md       # 可选
│       ├── {{内容标题}}-选题.md
│       ├── {{内容标题}}-大纲.md
│       ├── {{内容标题}}-初稿.md
│       ├── {{内容标题}}-终稿.md
│       ├── {{内容标题}}-复盘.md
│       └── 多平台分发/                # 可选
│       ├── 小红书.md
│       └── 知乎.md
```

**目录创建规则：**
- 使用当前年月作为一级目录（如 2026-03）
- 使用内容标题作为二级目录
- 标题从终稿的 `title` 字段提取
- 如果目录已存在，提示用户确认是否覆盖

### 步骤3：复制和处理每个文件

**对每个文件执行以下操作：**

1. **复制到归档目录**
   - 保持原文件内容
   - 重命名为标准格式：`{{标题}}-{{阶段}}.md`

2. **更新 YAML frontmatter**
   ```yaml
   ---
   title: "{{原标题}}"
   stage: 已归档           # 修改
   status: 已归档          # 修改
   archive_date: "{{DATE:YYYY-MM-DD}}"  # 新增
   original_path: "{{原文件路径}}"      # 新增
   # ... 其他字段保持不变
   ---
   ```

3. **添加归档标记**
   在文件末尾添加：
   ```markdown
   ---
   *归档时间：{{DATE:YYYY-MM-DD}}*
   *原路径：{{原文件路径}}*
   ```

### 步骤4：创建索引文件

**位置：** `06-已发布归档/YYYY-MM/{{内容标题}}-索引.md`

**内容模板：**
```markdown
---
title: "{{内容标题}}-内容链索引"
archive_date: "{{DATE:YYYY-MM-DD}}"
content_type: 索引
---

```

### 步骤6：多平台分发（可选）

如果用户提到有多平台版本：

1. **创建子目录：** `多平台分发/`
2. **复制各平台版本**（如有）
3. **在索引中添加章节：**
   ```markdown
   ## 多平台分发

   - [[{{内容标题}}-小红书.md]]
   - [[{{内容标题}}-知乎.md]]
   - [[{{内容标题}}-视频脚本.md]]
   ```

### 步骤7：删除原文件

**默认行为：归档后删除原文件**

归档完成后，删除原工作目录中的所有文件（包括选题、大纲、初稿、终稿、及各平台版本）。

**删除的文件列表：**
- 选题文件（如 `02-选题池/待写选题库/XXX.md`）
- 大纲文件（如 `03-内容工厂/1-大纲挑选区/XXX.md`）
- 初稿文件（如 `03-内容工厂/2-初稿打磨区/XXX-初稿.md`）
- 终稿文件（如 `03-内容工厂/3-终稿发布区/XXX-终稿.md`）
- 各平台版本（如 `05-图文工厂/小红书/XXX-小红书.md` 等）
- 灵感文件（如 `01-灵感与素材库/1-日常灵感剪报/XXX.md`）- 可选删除

**删除方式：**
使用 `rm` 命令删除原文件

**替代方案（如需保留原文件）：**
如果用户明确要求保留原文件（在归档时额外说明），则：
- 不删除原文件
- 在原文件YAML中添加 `archived_note: "已归档到 06-已发布归档/YYYY-MM/"`

## 注意事项

1. **归档前检查**：确保内容确实是已发布状态，避免归档未完成的内容
2. **目录命名**：使用内容标题作为目录名，避免使用过长或特殊字符
3. **文件完整性**：确保整个内容链都被正确复制和归档
4. **YAML更新**：归档后的文件必须更新stage和status字段
5. **索引准确性**：索引文件中的所有路径必须准确可访问
6. **复盘及时性**：鼓励用户在归档后立即填写复盘，趁记忆还新鲜
7. **多平台处理**：如果有多平台版本，确保都纳入归档
8. **原文件删除**：默认删除原工作目录中的文件，避免重复和积累
9. **归档目录组织**：按年月组织，方便后续查找和统计
10. **wikilinks验证**：确保所有归档文件的wikilinks正确指向归档目录内的文件
11. **删除确认**：删除前确认归档文件已正确创建，删除后提醒用户原文件已移除
