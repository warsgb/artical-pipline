# 内容推进 Skill

这是内容生产工厂的核心推进工作流：将当前内容自动推进到下一阶段，更新状态，建立回链，并在需要时创建下游文档。

## 触发条件

当用户出现以下任一情况时，触发此skill：

- 明确说"/推进"、"推进"、"推进到下一步"
- 说"/通过"、"通过审核"、"审核通过"
- 说"继续"、"开始下一步"
- 说"开始写这个选题"、"开始写大纲"、"开始写文章"
- 对某个文档说"推进到[阶段名]"

## 工作流程

### 步骤1：解析当前文档

**操作：**
1. 读取用户指定的文件（如果未指定，读取当前打开的文件）
2. 提取 YAML frontmatter 中的 `stage` 和 `status` 字段
3. 提取 `title`、`source_topic`、`source_outline` 等关联信息

**解析示例：**
```yaml
title: "AI写作方法论"
stage: 选题
status: 待写
source_topic: null
```

### 步骤2：确定下一阶段

根据**状态流转矩阵**确定目标阶段：

| 当前stage | 当前status | 触发动作 | 更新后stage | 更新后status | 是否创建下游 |
|-----------|-----------|---------|------------|------------|------------|
| 灵感 | 任意 | 推进 | 灵感 | 已转化 | ❌ |
| 选题 | 待写 | 推进 | 选题 | 进行中 | ❌ |
| 选题 | 进行中 | 提交修改 | 选题 | 待完善 | ❌ |
| 选题 | 待完善 | 通过 | 选题 | 已通过 | ❌ |
| 选题 | 已通过 | 继续 | 大纲 | 待完善 | ✅ 创建大纲 |
| 大纲 | 待完善 | 完成 | 大纲 | 已通过 | ❌ |
| 大纲 | 已通过 | 继续 | 初稿 | 写作中 | ✅ 创建初稿 |
| 初稿 | 写作中 | 提交修改 | 初稿 | 待完善 | ❌ |
| 初稿 | 待完善 | 通过 | 初稿 | 已通过 | ❌ |
| 初稿 | 已通过 | 继续 | 终稿 | 终稿待发布 | ✅ 创建终稿 |
| 终稿 | 终稿待发布 | 发布 | 终稿 | 已发布 | ❌ |
| 终稿 | 已发布 | 归档 | **触发bo-archive** | - | ✅ |

**默认推进逻辑：**
- 如果 `status` 是「进行中/写作中」→ 更新为「待完善」
- 如果 `status` 是「待完善」→ 更新为「已通过」
- 如果 `status` 是「已通过」→ 推进到下一阶段 + 创建下游文档
- 如果 `status` 是「待写」→ 更新为「进行中」

### 步骤3：执行状态更新

**更新当前文档的 YAML：**
```yaml
---
title: "{{原有title}}"
stage: "{{新stage}}"           # 根据矩阵更新
status: "{{新status}}"         # 根据矩阵更新
update_date: "{{DATE:YYYY-MM-DD}}"  # 新增
advance_note: "{{推进原因}}"    # 新增，如"审核通过，推进到大纲阶段"
# ... 其他字段保持不变
---
```

**advance_note 示例：**
- "选题审核通过，准备开始写大纲"
- "大纲已完成，准备生成初稿"
- "初稿修改完成，进入终稿阶段"

### 步骤4：建立下游文档（如需要）

**场景A：选题 → 大纲**

当从「选题-已通过」推进时：

1. **创建大纲文件**
   - 位置：`03-内容工厂/1-大纲挑选区/YYYY-MM-DD-{{title}}.md`
   - 使用模板：`[[90-系统配置/模板/大纲模板]]`
   - 填充字段：
     ```yaml
     ---
     title: "{{选题title}}"
     stage: 大纲
     status: 待完善
     source_topic: "[[02-选题池/2-已立项/原选题文件]]"
     create_date: "{{DATE:YYYY-MM-DD}}"
     update_date: "{{DATE:YYYY-MM-DD}}"
     ---
     ```

2. **更新选题的关联笔记**
   在原选题文件的「关联笔记」部分添加：
   ```markdown
   ## 关联笔记
   - 灵感来源：[[...]]
   - 已生成大纲：[[03-内容工厂/1-大纲挑选区/YYYY-MM-DD-XXX.md]]
   ```

**场景B：大纲 → 初稿**

当从「大纲-已通过」推进时：

1. **创建初稿文件**
   - 位置：`03-内容工厂/2-初稿打磨区/YYYY-MM-DD-{{title}}-初稿.md`
   - 使用模板：`[[90-系统配置/模板/文章模板]]`
   - 填充字段：
     ```yaml
     ---
     title: "{{大纲title}}"
     stage: 初稿
     status: 写作中
     source_outline: "[[03-内容工厂/1-大纲挑选区/原大纲文件]]"
     create_date: "{{DATE:YYYY-MM-DD}}"
     update_date: "{{DATE:YYYY-MM-DD}}"
     ---
     ```

2. **更新大纲的关联笔记**
   ```markdown
   ## 关联笔记
   - 选题来源：[[...]]
   - 已生成初稿：[[03-内容工厂/2-初稿打磨区/YYYY-MM-DD-XXX-初稿.md]]
   ```

**场景C：初稿 → 终稿**

当从「初稿-已通过」推进时：

1. **创建终稿文件**
   - 位置：`03-内容工厂/3-终稿发布区/YYYY-MM-DD-{{title}}-终稿.md`
   - 使用模板：`[[90-系统配置/模板/文章模板]]`
   - 填充字段：
     ```yaml
     ---
     title: "{{初稿title}}"
     stage: 终稿
     status: 终稿待发布
     source_outline: "[[03-内容工厂/2-初稿打磨区/原初稿文件]]"
     create_date: "{{DATE:YYYY-MM-DD}}"
     update_date: "{{DATE:YYYY-MM-DD}}"
     ---
     ```

2. **更新初稿的关联笔记**
   ```markdown
   ## 关联笔记
   - 大纲来源：[[...]]
   - 已生成终稿：[[03-内容工厂/3-终稿发布区/YYYY-MM-DD-XXX-终稿.md]]
   ```

### 步骤5：特殊场景处理

**场景D：终稿发布**

当用户对终稿说"/发布"时：

1. 更新 YAML：
   ```yaml
   stage: 终稿
   status: 已发布
   publish_date: "{{DATE:YYYY-MM-DD}}"  # 新增发布时间
   publish_platform: "{{用户指定的平台}}"  # 新增发布平台
   ```

2. 提示用户是否需要归档

**场景E：归档触发**

当 stage=终稿 且 status=已发布 且 用户说"归档"时：
- 转发到 **bo-archive** skill 处理

## 输出要求

完成后，向用户报告：

1. **状态更新信息**
   ```
   ✅ 已将「{{title}}」从 {{旧stage}}-{{旧status}} 推进到 {{新stage}}-{{新status}}
   ```

2. **下游文档创建（如适用）**
   ```
   📄 已创建：{{新文件路径}}
   - Stage: {{新stage}}
   - Status: {{新status}}
   - 已建立回链到原文件
   ```

3. **下一步提示**
   - 如果创建了大纲：提示"可以开始完善大纲结构"
   - 如果创建了初稿：提示"可以开始按大纲写作"
   - 如果创建了终稿：提示"终稿待发布，完成发布后可以归档"
   - 如果只是状态更新：提示"当前状态：{{新status}}，可以继续推进"

## 使用示例

### 示例1：选题推进到大纲

**用户输入：**

（当前打开的文件是选题，status=已通过）

```
- Status: 待完善
- 已建立回链到原选题文件

📌 下一步：可以开始完善大纲结构，完成后再次/推进
```

## 注意事项

1. **文件路径准确性**：创建下游文档时，确保路径符合工作流规则
2. **YAML格式正确**：更新frontmatter时，保持格式正确，不要破坏现有字段
3. **回链完整性**：每次创建下游文档，必须在上游文档中添加回链
4. **状态流转顺序**：严格按照状态流转矩阵执行，不要跳过阶段
5. **advance_note**：每次推进都应填写推进备注，方便后续追踪
6. **文件命名**：使用日期+标题的命名格式，保持一致性
7. **模板使用**：创建新文档时，必须使用对应的模板文件
